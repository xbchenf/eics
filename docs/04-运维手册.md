# EICS 企业智能客服系统 — 运维手册

> 版本：V2.0 | 最后更新：2026-06-28

---

## 1. 日常运维

### 1.1 服务状态检查

```bash
# 所有容器运行状态
docker compose ps

# 各组件健康检查
curl http://localhost:8080/api/v1/health       # Java 后端
docker exec eics-mysql mysqladmin ping -u root -p${MYSQL_ROOT_PASSWORD}  # MySQL
docker exec eics-redis redis-cli -a ${REDIS_PASSWORD} ping                # Redis
curl http://localhost:19530/healthz             # Milvus
curl http://localhost:9000/minio/health/live    # MinIO
```

### 1.2 日志查看

```bash
# 查看所有服务日志
docker compose logs -f --tail=50

# 查看单个服务日志
docker compose logs -f eics-server      # 后端日志
docker compose logs -f mysql            # 数据库日志

# 后端应用日志关键搜索
docker compose logs eics-server | grep "ERROR"
docker compose logs eics-server | grep "SLA告警"
```

### 1.3 资源监控

```bash
docker stats                    # 容器资源使用
df -h                           # 磁盘空间
free -h                         # 内存使用
```

关键指标告警阈值：

| 指标 | 告警值 | 处理 |
|------|--------|------|
| MySQL 数据目录 | > 80% | 清理历史日志或扩容 |
| MinIO 存储 | > 80% | 清理无用文件或扩容 |
| Java 堆内存 | > 80% | 调整 -Xmx 参数 |

## 2. 备份与恢复

### 2.1 数据库备份

```bash
# 每日全量备份
docker exec eics-mysql mysqldump -u root -p${MYSQL_ROOT_PASSWORD} \
  --single-transaction --routines --triggers \
  eics_main > /backup/eics_$(date +%Y%m%d).sql

# 建议 crontab 自动执行
0 3 * * * /opt/eics/backup.sh
```

### 2.2 数据库恢复

```bash
docker exec -i eics-mysql mysql -u root -p${MYSQL_ROOT_PASSWORD} \
  eics_main < /backup/eics_20260628.sql
```

### 2.3 备份范围

| 数据 | 备份方式 | 频率 |
|------|---------|------|
| MySQL | mysqldump | 每日 |
| MinIO 文件 | 备份 /minio_data 目录 | 每日 |
| 配置文件 | 备份 .env + docker-compose.yml | 变更时 |

## 3. 故障排查

### 3.1 服务无法启动

```bash
# 检查端口占用
netstat -tlnp | grep -E "80|8080|3306|6379"

# 查看启动日志
docker compose logs mysql | tail -50
docker compose logs eics-server | tail -50
```

### 3.2 数据库连接失败

1. 确认 MySQL 容器运行：`docker compose ps mysql`
2. 检查 `.env` 中的密码配置是否正确
3. 确认 `eics-server` 容器能 ping 通 `mysql`：`docker exec eics-server ping mysql`

### 3.3 LLM 调用失败

症状：知识库问答返回空或超时

1. 检查 DashScope API Key 是否过期
2. 检查网络是否能访问 `dashscope.aliyuncs.com`
3. 查看后端日志：`docker compose logs eics-server | grep -i "llm\|error"`

### 3.4 向量检索无结果

症状：上传文档后 AI 问答无相关回答

1. 确认文档状态为「就绪」：管理后台知识库页面查看
2. 检查 Milvus 健康：`curl http://localhost:19530/healthz`
3. 重启 Milvus：`docker compose restart milvus`

### 3.5 文件上传失败

1. 检查 MinIO 健康：`curl http://localhost:9000/minio/health/live`
2. 登录 MinIO Console（`http://localhost:9002`）确认 Bucket 存在
3. 检查磁盘空间：`df -h`

## 4. 性能优化

### 4.1 JVM 调参

编辑 `docker/docker-compose.yml` 中 `eics-server` 的 `JAVA_OPTS`：

```yaml
JAVA_OPTS: "-Xms512m -Xmx1024m -XX:+UseG1GC"
```

### 4.2 MySQL 优化

在 `docker/mysql/conf.d/` 下添加 `custom.cnf`：

```ini
[mysqld]
innodb_buffer_pool_size = 512M
max_connections = 200
slow_query_log = 1
long_query_time = 2
```

### 4.3 Milvus 索引优化

系统默认使用 HNSW 索引。大量文档场景可调整参数，参考 [Milvus 官方文档](https://milvus.io/docs)。

## 5. 应急预案

### 5.1 服务宕机

```bash
# 全部重启
docker compose down && docker compose --env-file .env up -d

# 单服务重启
docker compose restart eics-server
docker compose restart mysql
docker compose restart redis
```

### 5.2 磁盘满

1. 清理 Docker 无用镜像和容器：`docker system prune -a`
2. 清理 Milvus 日志：`docker exec eics-milvus rm -f /var/lib/milvus/logs/*.log.*`
3. 扩容或迁移数据目录

### 5.3 被入侵

1. 立即修改所有密码（`.env` 文件 + 数据库管理员密码）
2. 检查数据库是否有异常账号
3. 从备份恢复干净数据
4. 升级防火墙规则

## 6. 版本升级

```bash
# 1. 备份数据
./backup.sh

# 2. 拉取新版本
tar -xzf eics-v2.x.tar.gz

# 3. 停止旧版
docker compose down

# 4. 启动新版（保留数据卷）
docker compose --env-file .env up -d

# 5. 验证
curl http://localhost:8080/api/v1/health
```

> 数据卷（`*_data`）不会因 `down` 被删除，除非使用 `down -v`。
