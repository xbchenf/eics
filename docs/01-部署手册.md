## 2. 前置准备

### 2.1 服务器准备

以全新安装的 Ubuntu 22.04 LTS 为例：

```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 设置时区
sudo timedatectl set-timezone Asia/Shanghai

# 3. 创建部署目录
sudo mkdir -p /opt/eics
sudo chown $USER:$USER /opt/eics
```

### 2.2 安装 Docker

```bash
# 卸载旧版本（如有）
sudo apt remove docker docker-engine docker.io containerd runc

# 安装依赖
sudo apt install -y ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 添加 Docker 仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker Engine + Compose 插件
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 验证安装
docker --version        # 应输出 24.x+
docker compose version  # 应输出 v2.x+

# 将当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER
newgrp docker
```

### 2.3 配置防火墙

```bash
# 仅开放必要端口
sudo ufw allow 22/tcp     # SSH
sudo ufw allow 80/tcp     # Web 前端
sudo ufw enable
sudo ufw status
```

> ⚠️ 数据库端口（3306, 6379, 19530, 9000）**不要**对外暴露，仅容器间通信。

### 2.4 检查系统资源

```bash
# 确认内存 ≥ 8GB
free -h

# 确认磁盘 ≥ 50GB
df -h

# 确认 Docker 可用
docker run hello-world
```

## 3. 部署步骤

### 3.1 获取部署包

**方式一：从源码构建（推荐）**

```bash
# 1. 获取源码
git clone <仓库地址> /opt/eics/EICS-V2.0
cd /opt/eics/EICS-V2.0

# 2. 项目结构说明
#   EICS-V2.0/
#   ├── docker/                    ← 部署相关（Compose、SQL 初始化脚本、环境变量模板）
#   │   ├── docker-compose.yml    ← 服务编排文件
#   │   ├── .env.example          ← 环境变量模板
#   │   └── mysql/init/           ← 数据库建表脚本
#   ├── eics-server/              ← Java 后端源码
#   ├── eics-web/                 ← Vue 前端源码
#   └── docs/                     ← 项目文档

# 3. 构建后端
cd /opt/eics/EICS-V2.0/eics-server
# Linux/Mac: ./mvnw clean package -DskipTests
# Windows: mvn clean package -DskipTests
java -jar target/eics-server-v2-2.0.0-SNAPSHOT.jar  # 验证启动正常后 Ctrl+C 停止

# 4. 构建前端
cd /opt/eics/EICS-V2.0/eics-web
npm install
npm run build   # 产物: dist/

# 5. 回到部署目录
cd /opt/eics/EICS-V2.0/docker
```

**方式二：使用预编译发布包**

联系项目维护者获取 `eics-v2.0-release.tar.gz`，其中包含：

| 内容 | 说明 |
|------|------|
| `docker/` | docker-compose.yml、.env.example、SQL 初始化脚本 |
| `eics-server/target/*.jar` | 已编译的 Java 可执行 JAR |
| `eics-web/dist/` | 已编译的前端静态文件 |
| `docs/` | 项目文档 |

```bash
tar -xzf eics-v2.0-release.tar.gz -C /opt/eics
cd /opt/eics/eics-v2.0-release/docker
```

### 3.2 部署目录说明

部署时，`docker/` 目录是启动入口，`docker-compose.yml` 编排全部服务，通过上下文路径引用同级目录的 JAR 和前端产物：

```
部署目录/
├── docker/
│   ├── docker-compose.yml      ← 启动入口（编排所有容器）
│   ├── .env                    ← 环境变量（从 .env.example 复制并修改）
│   └── mysql/init/             ← MySQL 首次启动时自动执行的建表脚本
├── eics-server/target/*.jar    ← 后端 JAR（Compose 构建镜像时 COPY）
└── eics-web/dist/              ← 前端静态文件（Compose 构建镜像时 COPY）
```

### 3.3 配置环境变量

```bash
cp .env.example .env
vim .env
```

关键配置项：

```ini
# MySQL
MYSQL_ROOT_PASSWORD=your_root_password
MYSQL_DATABASE=eics_main
MYSQL_USER=eics
MYSQL_PASSWORD=your_db_password

# Redis
REDIS_PASSWORD=your_redis_password

# MinIO (文件存储)
MINIO_ROOT_USER=admin
MINIO_ROOT_PASSWORD=your_minio_password

# Milvus 内部使用
MILVUS_ACCESS_KEY=minioadmin
MILVUS_SECRET_KEY=minioadmin

# LLM (阿里云 DashScope)
DASHSCOPE_API_KEY=sk-your-api-key
LLM_MODEL=qwen-max
EMBEDDING_MODEL=text-embedding-v3

# JWT
JWT_SECRET=your-jwt-secret-key-change-in-production
JWT_EXPIRE_HOURS=72
```

### 3.4 启动所有服务

```bash
docker compose --env-file .env up -d
```

启动顺序：MySQL → Redis → Milvus（含 etcd + MinIO）→ MinIO → Java 后端 → Nginx 前端

### 3.5 验证部署

```bash
# 检查所有容器运行状态
docker compose ps

# 确认后端健康
curl http://localhost:8080/api/v1/health

# 前端访问
浏览器打开 http://localhost
```

## 6. 配置 LLM

系统依赖大语言模型提供智能问答和实体提取能力。支持任何兼容 OpenAI 协议的 API。

### 阿里云 DashScope（推荐）

1. 登录 [阿里云 DashScope 控制台](https://dashscope.console.aliyun.com/)
2. 开通模型服务，获取 API Key
3. 将 API Key 填入 `.env` 中 `DASHSCOPE_API_KEY`

### 其他兼容服务

修改 `docker/eics-server/Dockerfile` 中的环境变量：

```dockerfile
ENV SPRING_AI_OPENAI_API_KEY=${DASHSCOPE_API_KEY}
ENV SPRING_AI_OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

## 8. 常见问题

| 问题 | 排查 |
|------|------|
| 启动失败 | `docker compose logs mysql` 查看数据库初始化日志 |
| 无法访问前端 | 确认 80 端口未被占用：`netstat -tlnp \| grep 80` |
| LLM 调用失败 | 检查 `.env` 中 `DASHSCOPE_API_KEY` 是否正确 |
| 向量检索无结果 | 确认 Milvus 健康：`curl http://localhost:9091/healthz` |
| 文件上传失败 | 检查 MinIO：访问 `http://localhost:9002` 用 `MINIO_ROOT_USER/PASSWORD` 登录 |

## 9. 离线部署（无 Docker）

如需传统部署方式：

1. 安装 JDK 17、MySQL 8.0、Redis 7、Milvus 2.4、MinIO
2. 执行 `docker/mysql/init/01-init-tables.sql` 初始化数据库
3. 修改 `application.yml` 中的数据库/Redis/Milvus/MinIO 连接地址
4. 启动后端：`java -jar eics-server-v2-2.0.0-SNAPSHOT.jar`
5. 前端静态文件部署到 Nginx，配置反向代理到后端 8080 端口
