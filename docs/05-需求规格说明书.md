# EICS 企业智能客服系统 — 需求规格说明书

> 版本：V2.0 | 最后更新：2026-06-28

---

## 1. 引言

### 1.1 项目背景

企业内部 IT 支持和客户服务场景中，存在大量重复性咨询工作。传统方式依赖人工逐一回复，效率低、响应慢、无法 7×24 小时服务。EICS 系统通过 AI 智能问答 + 工单管理 + 人工坐席协作，解决这一痛点。

### 1.2 产品定位

企业级智能客服平台，私有化部署，数据不外传。面向中大型企业的 IT 服务台、行政服务中心等场景。

### 1.3 用户角色

| 角色 | 说明 | 典型用户 |
|------|------|---------|
| USER | 普通用户 | 全体员工 |
| AGENT | 人工坐席 | IT 支持人员、客服 |
| ADMIN | 系统管理员 | IT 管理员 |

### 1.4 术语定义

| 术语 | 说明 |
|------|------|
| RAG | 检索增强生成：从知识库检索相关文档 → LLM 生成答案 |
| 向量检索 | 将文档转为数学向量，通过语义相似度匹配 |
| 工单 | 用户提交的服务请求，含问题描述、优先级、SLA 等 |
| SLA | 服务等级协议：工单必须在规定时间内响应 |
| 会话 | 用户与系统（或坐席）的一次实时文字对话 |

## 2. 功能需求

### 2.1 智能问答 (RAG)

**FR-01** 系统应支持用户以自然语言提问，从知识库中检索相关文档并生成答案。

- 支持文档格式：PDF、Word (.docx)、TXT、Markdown
- 答案应附带参考文档溯源
- 当检索无结果时，自动提示转人工

### 2.2 多轮对话工单

**FR-02** 系统应引导用户通过多轮对话填写工单信息，替代传统表单。

**流程**：发起 → 输入手机号 → 选择问题类型 → 描述故障 → 提交

**问题类型**：硬件故障、网络问题、权限申请、软件安装、其他

**FR-03** 系统应自动校验手机号格式（11 位，1 开头）

**FR-04** 系统应根据问题类型自动设定优先级：

| 问题类型 | 优先级 | SLA |
|---------|--------|-----|
| 硬件故障 | P1 | 2 小时 |
| 网络问题 | P1 | 2 小时 |
| 权限申请 | P3 | 8 小时 |
| 其他 | P2 | 8 小时 |

### 2.3 人工坐席服务

**FR-05** 用户可通过按钮或输入关键词发起转人工请求

**FR-06** 坐席应能看到待接入用户列表，点击接入开始服务

**FR-07** 坐席与用户通过 WebSocket 实时文字通信

**FR-08** 坐席应可使用预设快捷回复

**FR-09** 坐席关闭会话后，系统应向用户推送服务评价邀请

### 2.4 工单管理

**FR-10** 管理员和坐席可查看、搜索、筛选所有工单

**FR-11** 坐席可认领工单（PENDING → PROCESSING）

**FR-12** 坐席可解决工单（PROCESSING → RESOLVED）

**FR-13** 坐席可手动调整工单优先级

**FR-14** 系统自动计算 SLA 截止时间，超时工单实时告警

**FR-15** 用户可在聊天页面查看自己的工单列表

### 2.5 知识库管理

**FR-16** 管理员可上传文档并指定分类和标签

**FR-17** 系统自动解析文档内容并建立向量索引

**FR-18** 管理员可修改文档分类、标签

**FR-19** 管理员可删除文档

**FR-20** 支持按分类筛选文档

### 2.6 用户与权限

**FR-21** 支持用户自助注册（默认 USER 角色）

**FR-22** 管理员可创建、启用/禁用、修改用户角色

**FR-23** 坐席和管理员登录后自动上线，退出自动下线

### 2.7 满意度评价

**FR-24** 人工服务结束后系统推送 1-5 星评分

**FR-25** 用户可填写选填留言

**FR-26** 同一会话不允许重复评价

**FR-27** 管理员可在工作台查看满意度统计

### 2.8 工作台

**FR-28** 工作台展示系统运行关键指标（今日会话、待处理工单、在线坐席数、满意度等）

**FR-29** 展示工单优先级分布图和 7 日趋势图

**FR-30** 展示最近会话和工单列表

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 |
|------|------|
| 智能问答响应 | P95 < 5 秒 |
| 普通消息响应 | < 500 毫秒 |
| 并发用户 | 支持 100 并发 |

### 3.2 可用性

- 核心服务可用性 ≥ 99.5%
- 计划内维护窗口 < 每月 4 小时

### 3.3 安全

- 密码 BCrypt 加密存储
- JWT Token 认证，72 小时过期
- 用户消息脱敏处理（手机号中间四位屏蔽）
- 管理端接口需登录认证

### 3.4 可维护性

- Docker 一键部署
- 配置文件集中管理（.env）
- 日志统一输出到标准输出

### 3.5 兼容性

- 浏览器：Chrome 90+ / Edge 90+ / Firefox 90+
- 分辨率：≥ 1280×720

## 4. 业务流程图

### 4.1 用户服务主流程

```
用户提问
  ├─ AI 能解答 → RAG 知识库回答 → 结束
  ├─ 提交工单 → 填写信息 → 工单生成 → 坐席处理 → 解决 → 评价
  └─ 转人工 → 坐席接入 → 实时沟通 → 关闭会话 → 评价
```

### 4.2 工单生命周期

```
创建(PENDING) → 认领(PROCESSING) → 解决(RESOLVED) → 关闭(CLOSED)
```

## 5. 约束与假设

### 5.1 约束

- LLM 依赖外部 API（阿里云 DashScope），需外网访问
- 知识库答案质量依赖文档内容完整性和 LLM 能力

### 5.2 假设

- 用户具备基本的浏览器操作能力
- 企业提供稳定的内网环境
- 管理员负责知识库文档的及时更新
