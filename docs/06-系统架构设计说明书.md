# EICS 企业智能客服系统 — 系统架构设计说明书

> 版本：V2.0 | 最后更新：2026-06-28

---

## 1. 概述

### 1.1 设计目标

- **纯 Java 技术栈**：单一语言开发，降低运维复杂度
- **私有化部署**：所有数据存储在客户自有环境，不外传
- **弹性扩展**：核心服务支持水平扩展
- **高可用**：关键组件支持数据持久化和故障恢复

### 1.2 技术选型

| 层 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 后端框架 | Spring Boot | 3.3.x | 企业级 Java 标准框架 |
| JDK | Java | 17 LTS | 长期支持版本 |
| ORM | MyBatis-Plus | 3.5.x | 轻量级，Lambda 查询 |
| AI 编排 | Spring AI | 1.0 GA | 统一 LLM 调用抽象 |
| LLM | Qwen-Max (DashScope) | - | 中文理解能力强 |
| 向量数据库 | Milvus | 2.4 | HNSW 索引，高性能检索 |
| 缓存 | Redis | 7.x | 会话状态 + 在线状态 |
| 数据库 | MySQL | 8.0 | 成熟稳定 |
| 对象存储 | MinIO | latest | S3 兼容，私有部署 |
| 前端 | Vue 3 + Element Plus | 3.x | 组件化开发 |
| 构建 | Vite | 5.x | 快速开发构建 |
| 容器化 | Docker Compose | v2 | 一键部署 |

## 3. 核心设计

### 3.1 对话引擎

采用 **状态机主导 + LLM 增强** 的混合架构。

**设计原则**：
- 意图路由：关键词精确匹配优先（确定性），LLM 辅助分类兜底
- 槽位填充：Redis 持久化状态机驱动流程，LLM 辅助实体提取
- 降级保障：LLM 超时或失败时，正则表达式兜底

**状态机设计**：

```
IDLE → COLLECTING_PHONE → COLLECTING_ISSUE_TYPE → COLLECTING_DESC → DONE
                       ↑_____________________________|
                       任一步骤失败 → 重新追问
```

状态存储在 Redis `dialog:{sessionId}:state`，TTL 30 分钟自动清理。

### 3.2 RAG 检索增强生成

```
用户提问
  → Embedding 模型向量化
  → Milvus HNSW 相似度检索 (Top-K)
  → 召回文档片段 + 用户问题 → LLM Prompt
  → LLM 生成答案
  → 附带参考来源返回用户
```

### 3.3 WebSocket 通信

**路由策略**：
- 会话处于 BOT 模式 → 对话引擎处理 → 回复推回用户
- 会话处于 AGENT 模式 → 双向实时转发（用户 ↔ 坐席）

**安全**：AGENT 消息需携带有效 JWT Token

### 3.4 SLA 监控

`@Scheduled(fixedRate = 60000)` 每分钟扫描 PENDING 工单，发现超时实时 WebSocket 推送告警到所有在线坐席。已通知工单去重，避免重复告警。

### 3.5 认证与鉴权

- JWT Token 无状态认证，72 小时过期
- AuthInterceptor 拦截除白名单外的所有请求
- 前端 Axios 拦截器自动注入 Token，401 自动跳转登录

## 4. 部署架构

```
┌──────────────────────────────────────────────────┐
│                   Docker Host                     │
│  ┌─────────┐  ┌─────────┐  ┌──────────────────┐  │
│  │ Nginx   │  │ Java    │  │ 中间件              │  │
│  │ :80     │→ │ :8080   │→ │ MySQL :3306        │  │
│  │         │  │         │  │ Redis :6379        │  │
│  └─────────┘  └─────────┘  │ Milvus :19530     │  │
│                             │ MinIO  :9000/9002 │  │
│                             └──────────────────┘  │
└──────────────────────────────────────────────────┘
```

所有服务通过 `eics-net` 桥接网络通信，仅 Nginx (80) 和必要端口对外暴露。

## 5. 安全设计

| 层面 | 措施 |
|------|------|
| 传输 | 生产环境建议配置 HTTPS (Nginx) |
| 认证 | JWT Token，72h 过期 |
| 密码 | BCrypt 加盐哈希 |
| 数据脱敏 | 手机号 `138****1234` 中间四位屏蔽 |
| 输入过滤 | 敏感词过滤 |
| 接口鉴权 | AuthInterceptor 全局拦截 |
| Docker | 最小端口暴露，仅 80 / 8080 / 9002 |

---

## 6. 扩展性设计

- **水平扩展**：Java 后端为无状态服务，可部署多实例 + 负载均衡
- **Redis 集群**：支持 Redis Cluster 模式
- **MySQL 读写分离**：MyBatis-Plus 支持多数据源
- **Milvus 分布式**：支持从 Standalone 升级到 Cluster 模式
